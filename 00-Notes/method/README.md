# Method

Material that sharpens how the project works — external thinkers, local concepts, outward-facing explanations.

This is not a second patterns layer. Nothing here governs the repo. It clarifies live practice problems and, where needed, makes the project legible to others.

## Use

- File a note here when something helps articulate a repo method problem that the primary sources and existing apparatus do not yet name well.
- Keep notes bounded: what problem it sharpens, what it does not license, current encounter status.
- Do not promote to architecture. If a concept earns standing use, it migrates into `patterns/` through the normal promotion path.
