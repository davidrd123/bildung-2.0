# Collision-Ready Surfaces

Purpose: keep a lightweight locator for bounded source-side materials that are ready, or nearly ready, for cross-project collision work.

Status: living-layer locator only. This is not a claim that the listed contacts all hold. It is a sourcing aid so later agents do not have to reconstruct bounded candidate materials from chat history.

Use:

- start here when a bounded source-collision note is being considered
- prefer candidates with exact passage surfaces already named
- if a candidate has only theme-language and no bounded materials, it is not ready yet

Reading key:

- `collision-bundle-*` files are sourcing bundles: bounded materials, scopes, refusals, and handoff shapes
- `collision-*` files inside dated experiment folders are actual runs against one bundle or comparison question
- a bundle can exist before any successful run; a run should point back to its bundle when one exists
