# Experiments

This folder holds bounded speculative probes run against earned material.

It is still part of the living layer. It is not a second authority layer, and it is not a license to bypass source work.

See also:

- `live-options.md` for the current queue of candidate experiments and held prompts

## Use

Put a note here when the work is:

- a clearly bounded experiment
- using already earned source surfaces as testbeds
- stating its question, materials, and failure conditions explicitly
- intended to produce an honest result, including a useful negative

Do not use this folder for:

- ordinary translation support
- routine DeepResearch question sets or audits
- standing architecture
- speculative frameworks that have not yet been tested against the material

## Promotion Rule

An experiment note does not become a repo claim by being interesting.

Only promote what survives return to the relevant source surfaces.

## Current Method Discipline

Use the smallest control that can still answer the question.

### Packet construction

- match scale before you collide materials
- prefer earned local surfaces over broad thematic inventory
- keep the packet small enough that a negative remains informative
- name one sharp question before reading
- when an experiment is actually run and likely to be reused, preserve the as-run packet as a standalone packet note
- if the packet was not written in advance, reconstruct it immediately afterward while the loci and controls are still exact

### Middle-band collision notes

Use a bounded source-collision note when a question is too open for a strict replay packet but should not be left at chat-atmosphere level.

Controls:

- keep the source surfaces bounded and explicit
- allow the output question to stay more open than a packet's output contract
- require at least one resistance or refusal, not only a contact zone
- treat the note as workshop-floor material, not as a standing cross-project claim

Quality checks:

- strongest when the collision produces a surprise the prior conversation did not already script
- stronger still when a fresh reader, given the same bounded materials, converges on the same zone without sharing the wording
- weaker when the note mainly redescribes a result already produced by a stricter packet without adding a new usable adjacency

Use the collision note to locate the next tighter rerun, not to replace it.

### Result discipline

- require one constrained sentence
- require one useful negative where a tempting bridge fails
- include a short `what this does not license` section
- treat honest failure as a result, not as wasted work

### Source discipline

- if the claim is source-level, rerun it against the original language before trusting it
- use English drafts as checks when needed, not as the highest evidence
- do not widen from packet evidence to whole-author claims

### Independent check

When a result seems worth trusting beyond one local run:

- craft a standalone packet note
- hand only that packet and the source files to a fresh agent or reader
- do not include prior experiment prose in the handoff
- compare convergence, divergence, and especially the negative result

For middle-band collision notes, the analogous check is simpler:

- keep the same bounded materials
- hand them to a fresh reader with no preferred bridge sentence
- compare whether the same contact zone, resistance, and surprise survive

If the fresh read converges only at the level of atmosphere, tighten the surfaces. If it converges on a usable adjacency, the collision was doing real work.

If the packet over-steers the answer, tighten it. If the result survives an independent run, the control has improved.

### Critique closure

When a small experiment chain starts producing reusable bridge sentences:

- run one self-critique pass against the experiment prose itself
- preserve one sentence that survives as still warranted
- cut at least one overconstituted bridge if the packet justifies it
- rewrite the overreach into a stricter sentence instead of treating critique as total veto

This is a control on wording and warrant, not a ban on synthesis.

### Reading convergence

- same-packet convergence across different readers or models is useful evidence, not absolute proof
- the strongest sign of a good packet is often: shared core result, different informative negatives
- if every run agrees on the center but fails in different places, the material is probably resisting over-reading rather than merely echoing the packet
- if every run produces the same elegant bridge and no serious negative, assume the packet may be over-steering
