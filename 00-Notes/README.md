# 00-Notes

Living workspace for repo-level thinking, synthesis, and session carry-forward.

This folder is not a second authority layer. It is the exploratory layer described in `process.md`.

## Cold Start

Start here:

- `right-now.md` — standing snapshot: what is proven, current, next, and live
- `process.md` — workflow, authority order, ingestion, and upkeep

If you are orienting to a specific live line next, go from those files into the relevant subproject or note cluster.

## Top Level

The top level is reserved for standing or semi-standing repo notes:

- `right-now.md` — current state
- `process.md` — workflow and authority
- `higher-purpose-distillation.md` — project telos / instrument pressure
- `genealogy-to-instrument.md` — genealogy into architecture
- `cross-domain-synthesis-threads.md` — active cross-domain threads
- `symbiotic-vault-intersection.md` — vault / repo relationship

Do not treat dated notes at this level as standing state. Those now live under `distillations/`.

## Subdirectories

- `Chats/` — exported conversations; raw exploratory material
- `distillations/` — filtered notes extracted from chats or research runs; useful, but not standing state and not canonical architecture
- `DeepResearch/` — question sets, audits, prompt surfaces, and external pressure tests
- `experiments/` — bounded speculative probes run against earned material; explicitly testable and allowed to fail
- `cross-project/` — workspace-level synthesis notes between live source projects
- `threads/` — active thread dossiers when a thread earns a more stable home

## Use Rule

Use `00-Notes/` to:

- recover session context quickly
- hold exploratory synthesis without promoting it too early
- route from chats or research outputs into more disciplined notes

Do not use `00-Notes/` to override:

- `patterns/charter.md`
- `patterns/handle-schema.md`
- canonical subproject files

If a note here proposes something the canonical layer does not yet support, treat it as a proposal.
