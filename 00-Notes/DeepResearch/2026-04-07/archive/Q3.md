# Evidence Hooks By Claim

### Claim 5 — Journals should add greppable `[method]` or `[process]` tags for process findings

Evidence is **lightly present**, but the tag proposal itself is not yet live.

The underlying need is real. `patterns/charter.md` already says reviews should preserve findings about method and cross-batch patterns rather than leaving them in chat context, and it already permits one lightweight inline marker, `[visual candidate]`, when later recovery matters. `texts/erkenntnisproblem-vol1/journal.md` already contains a local “Method note,” and the addition of `reading/close-reading-ledger.md` was explicitly justified as a way to stop paragraph-scale gains from disappearing. So the repo clearly has a real process-trace problem in at least one prose-heavy zone.

What weakens the full claim is medium asymmetry. In `texts/zeitmauer/`, process and commentary are already partly retrievable through `source/handles.yaml`: journal notes, translation notes, thread handles, and evidence handles are addressable without journal tags. In the live files here, the specific `[method]` / `[process]` convention does not appear as an implemented practice. So the claim survives only in a weaker form: a retrieval aid for places where prose traces are being lost, not a repo-wide journal schema.

### Claim 6 — Certain modern or adjacent sources should be held in a positive quarantine state with specific sticks / controlled probes

Evidence is **lightly present**, and the repo already has a proto-version of this practice.

`sources/modern/README.md` already enforces demand-led intake and says a modern encounter earns its keep only when it changes a live reading. The queue entries for texts like Deacon and Whitehead already carry explicit “Trigger conditions,” which are functionally close to sticks: concrete questions or pressures that must be met before deeper encounter is justified. `sources/modern/incoming/anabology-everything-oscillates-together.md` already uses `encounter_status: located` plus a `reason`, which is already a real reserve state rather than a bare holding pile.

What is absent is a canonical `quarantined` status or a standing `poke_with` field. So the claim survives only in weaker form: keep risky sources in existing `located` / trigger-conditioned states and add explicit probe questions inside current note formats. The overclaim is the new status vocabulary, not the controlled-probe practice.

### Claim 7 — The repo should separate universal methodology from model-specific notes

Evidence is **lightly present**, but the full rhetoric of the claim is too strong.

The pressure is real because `CLAUDE.md` mixes durable project method with current-model and agent-facing cautions. Universal rules like evidence-before-framework, deferred structure, and do-not-reference-what-you-have-not-encountered sit beside model-specific warnings about LLM habits and Proteus-style behavior. That blend is real.

But the repo already has a durable methodological layer outside `CLAUDE.md`: `patterns/charter.md`, `00-Notes/process.md`, and `patterns/handle-schema.md` already carry project-level purpose, authority, review discipline, lifecycle language, and anti-overbuild constraints. So the surviving claim is narrower: the repo does not need a wholly new methodological layer; it needs the already-existing split made more legible where `CLAUDE.md` currently blurs it. This is a refactoring pressure, not yet an earned canonical file split.

### Claim 8 — A lightweight ligature pass is a plausible rhythm for noticing cross-project contact without overbuilding

Evidence is **already present in practice**, though not under that name.

`texts/zeitmauer/journal.md` already contains bounded cross-project work such as the “Predecided World” note, and that note explicitly keeps the cross-project branch subordinate to translated sections rather than letting it float free. `texts/zeitmauer/source/handles.yaml` already integrates that note as `note:journal:predecided-world`, and the same file already relates section anchors into thread dossiers. `texts/zeitmauer/threads/time-crisis.md` and `threads/predecided-world.md` show that higher-order connection work can already stay tied to passages and handles rather than becoming free framework.

`00-Notes/process.md` explicitly says cross-project and modern-source contact traces are still the weakest interoperability point, and `00-Notes/right-now.md` names essentially the same issue as a live question. So this claim survives fairly well, but only in bounded form: a connection-only pass using existing homes. The premature part is any move to turn “ligature pass” into a new object type, new standing file, or fixed mandatory rhythm.

### Claim 9 — A periodic metabolic review is a plausible rhythm for deciding whether the repo’s current trace media are keeping up

Evidence is **lightly present**.

`00-Notes/process.md` already contains a “Secondary Trace Diagnostic” that compares Cassirer’s prose-trace environment with Zeitmauer’s relational one and draws repo-level conclusions: do not force uniformity of format, do not demote `handles.yaml` to mere navigation where it is already a real trace surface, and do make cross-project / modern-source contact traces lighter and more routine. Functionally, that is already a prose metabolic review.

What is missing is a repeated non-quantitative practice that has become a real repo rhythm. `right-now.md` adds live questions and hard anti-overbuild constraints, including “Do not build a dashboard,” which cuts directly against any inflated version of this claim. With the pulse / metrics layer explicitly unearned, the claim survives only in weakened form: occasional prose diagnosis when trace pressure becomes visible, not a standing regulatory layer.

# Smallest Viable Tests

### Claim 5 — `[method]` / `[process]` tags

Run a pilot in one prose-heavy site only, most plausibly `texts/erkenntnisproblem-vol1/journal.md`. For the next few sessions, mark only genuine process findings that would otherwise vanish. Then check, on a later cold re-entry, whether those notes are easier to recover than the current untagged method material. Do not universalize the convention unless it solves a real retrieval problem that the current ledger/journal combination does not already solve.

### Claim 6 — Quarantine with sticks / controlled probes

Take one already-`located` modern source and add two or three explicit probe questions inside its existing note or queue entry, without inventing a new schema. Then do one controlled probe against one live anchor: one chapter or section, one question, one short record of whether the source changed a live reading or merely confirmed adjacency. If it changes nothing, leave it where it is. If it changes a live reading, let the existing modern-source workflow absorb it.

### Claim 7 — Methodology vs model-notes split

Make a temporary sorting pass over `CLAUDE.md` in a living note, not in canonical files. Separate each rule into either durable project method or current model / agent finding. The test is whether the durable side mostly resolves back to `charter.md`, `process.md`, and `handle-schema.md`, while the empirical side can change without touching those files. If that sorting holds cleanly, later crystallization is earned.

### Claim 8 — Ligature pass

After the next few primary work sessions, do one connection-only pass using existing homes: a short cross-project journal note, a thread update, or a few new relations in `handles.yaml`. Cap it hard. Record only surprising contacts, and require each one to name concrete anchors on both sides. If the pass yields only expected echoes, or expands into free synthesis, it has not earned a standing rhythm.

### Claim 9 — Metabolic review

Repeat the `Secondary Trace Diagnostic` once, in prose, when the next trace-pressure question becomes live. Use only questions the repo already knows how to answer: can a cold start reconstruct why a local decision was made, can cross-project or modern-source contact be found from both sides, and are any trace media obviously failing? Record one judgment in `process.md` or `right-now.md` and stop. No metrics, no dashboard, no new layer.

# Claims That Need More Than The Repo Has

### Claim 5 in its strong form

A repo-wide `[method]` / `[process]` convention needs more than the current repo shows. The live evidence supports a prose-heavy pilot, not a universal rule across deliberately different trace media.

### Claim 6 in its strong form

A canonical `quarantined` status with a standing `poke_with` schema needs more than the repo currently has. The repo already supports `located`, `reason`, and trigger conditions. That is enough to test controlled probes. It is not yet enough to justify a new lifecycle status.

### Claim 7 in its strong form

An immediate split into something like `METHODOLOGY.md` and `MODEL-NOTES.md` needs more evidence than the repo currently provides. The distinction is real, but the durable methodology layer already exists across `charter.md`, `process.md`, and `handle-schema.md`. What is not yet shown is that two new canonical files would solve a real working problem better than clarifying the overlap in `CLAUDE.md`.

### Claim 9 in its strong form

A standing metabolic-review program needs more than the repo has. One good diagnostic pass does not yet justify cadence, generalized criteria, or quasi-regulatory review procedure. In particular, anything that smuggles back in orphan audits, handle vitality measures, or cross-repo health metrics is blocked by the present state of the repo and by the explicit anti-dashboard constraints.

### Claim 8 is the main exception

In weakened form, it does **not** need more than the repo has. The repo already has the right homes, the right pressure, and at least one strong pilot for bounded connection work.
