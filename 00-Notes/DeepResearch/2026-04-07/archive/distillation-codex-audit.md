## 2) Promising residue not yet adequately kept

The biggest under-kept residue is the narrow translation-resistance method from the Planmäßigkeit/Uexküll cluster. The extraction note already states the real question cleanly: “Is translation resistance here a bug to solve or evidence to preserve?” It also records modest next moves: add a “what did the resistance reveal?” field, watch for conceptual false friends, try one morphological-bridging pass as experiment, and treat the German term as a first-class annotation beside provisional English (`00-Notes/Chats/2026-04-06/extractions/2026-04-06_actual-questions-from-planmaessigkeit-uexkuell-chat.md`). The April 7 source-role notes do a good job with bibliographic routing, but they do not adequately carry forward this smaller, more text-proximate residue. Under repo standards, this is exactly the kind of thing that should be kept locally and lightly: it is source-pressure, not framework import; it fits `CLAUDE.md`’s “Do not reference what you have not encountered”; and it matches `00-Notes/process.md`’s advice to record retrieval-visible distinctions compactly rather than inventing new architecture. A small Uexküll/translation-method distillation is still missing.

Second, the repo now has permission to capture raw formation traces, but not yet a clear maturation path for them. The session-continuity note says raw `formation` traces may be useful “if they stay optional and reviewable” (`00-Notes/distillations/2026-04-07-earned-distillation-from-warburg-mythos-chats-on-session-continuity.md`). The Warburg method-pressure note separately keeps `[method]` or `[process]` markers only as a productive prompt (`00-Notes/distillations/2026-04-07-earned-distillation-from-warburg-chat-on-method-pressure.md`). Meanwhile `patterns/charter.md` says that if a review surfaces a cross-batch pattern or a finding about method, “write it down” in the relevant journal, and `00-Notes/process.md` says reviews should capture patterns and method findings that the original session could not yet see. What is still under-kept, then, is not more capture, but a tiny maturity rule: when does a raw ledger trace die, when does it stay local, and when does it graduate into a journal review note? Right now the pieces exist, but the path between them is only implicit.

Third, controlled probe is now admitted as a real missing mode, but it is not yet operationally coupled to the source-role routing material. The Warburg method note defines the mode well — “one chapter or bounded slice / one explicit question / one note on whether the source resisted, taught, or merely confirmed” — and explicitly says it should stay a living-layer practice, not a new canonical ontology (`00-Notes/distillations/2026-04-07-earned-distillation-from-warburg-chat-on-method-pressure.md`). The source-role distillation and tag note, meanwhile, classify sources by role but do not yet say when a category-2, -3, or -4 item should be merely located, probed, or fully encountered (`00-Notes/distillations/2026-04-07-earned-distillation-from-source-role-taxonomy-chat-and-deepresearch.md`; `00-Notes/distillations/2026-04-07-source-role-tags-for-april-6-deepresearch.md`). `sources/modern/README.md` already supplies trigger discipline and `status: open`, so this does **not** need a new canonical status system. But there is enough residue for one tiny living-layer routing rule that links “located,” “controlled probe,” and “full encounter.” That residue is promising and not yet adequately kept.

If I reduce the audit to one sentence: the April 7 notes are good at filtration and scale control, but the still-missing keep is not bigger architecture — it is two small operational residues: the translation-resistance method from the Uexküll cluster, and the maturation path from raw session/process traces into journal-worthy method findings.
