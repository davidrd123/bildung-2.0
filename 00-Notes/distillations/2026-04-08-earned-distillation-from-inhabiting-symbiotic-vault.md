## Promoted claims

### 1. The main gain is not new ontology but sharper operating conventions

The Symbiotic-Vault visit did not show that `bildung-2.0` is missing an atom layer, a merged frame system, or a whole new pipeline. The repo already has a usable layer split, a stable handle grammar, real promotion gates, and multiple trace media matched to different textual grain.

What the vault made more visible is that some agent-facing conventions are still implicit:

- what a specific artifact is allowed to claim
- how session continuity should be preserved
- where freer synthesis belongs relative to source-bound notes
- when a bounded structural scan is licensed

This keeps the lesson at the level of operation, not ontology.

### 2. Artifact-specific handoffs are a real missing sharpener

The bounded movement-scan handoff was useful because it constrained the epistemics of one artifact type rather than repeating general project virtues. It named permissible and impermissible claims, separated candidate pressure from pressure worth caring about, and forced an honesty section into the output.

`bildung-2.0` already has strong general guardrails in `patterns/charter.md`, `00-Notes/process.md`, and `CLAUDE.md`. The next gain looks narrower: handoff-style constraints for artifact types that already exist here, especially:

- thread dossiers
- cross-project synthesis notes
- bounded reread memos
- promotion scans on local apparatus

The lesson is not "write more prompts." It is "make artifact epistemics explicit where overreach is predictable."

### 3. The Cassirer session ledger is the strongest current pilot for agent memory

The vault's `_memory` layer made it easier to see what is already working in `erkenntnisproblem-vol1`: the session ledger preserves state on entry, what shifted, unresolved pressure, re-entry instructions, and optional formation notes in a compact reusable form.

That is already more than a recap surface. It is a live pilot for the repo's still-missing agent-memory conventions. The right move is not to invent a repo-wide memory architecture in the abstract. It is to treat the Cassirer ledger as the proving ground and promote only what survives reuse there and in at least one second subproject.

This also suggests a narrower next experiment: when a repeated trigger becomes locally real, prefer a small trigger -> action watchpoint in the subproject ledger over a repo-wide rule.

### 4. `bildung-2.0` already has the right split between source-bound and freer surfaces, but should use it more deliberately

The vault's raw / structured distinction helped make visible that Cassirer already has a cleaner split than some live sessions remember:

- `journal.md` is source-bound and workflow-bound
- `interpretive-notes.md` is where analogies, cross-project connections, and method extrapolations belong
- `formation_notes` in the session ledger preserve genuine shifts in grip without silently hardening them into framework

This means the repo does not need a new `Candid` layer imported whole. It does need to keep using the surfaces it already has according to their actual roles, especially when review produces tempting extrapolations.

### 5. The distinction between tracking threads and reading frames is useful in the living layer

The vault's frame-read work made a real distinction easier to see:

- some structures primarily track recurrence or traversal
- others actively read the material through a perspective and therefore highlight some things while hiding others

`bildung-2.0` already has both behaviors, but they are not always named separately. The Goethe / Leibniz split, the Schuon limit, and similar method lenses are doing frame-like work. The Mumford bridge, Anders bridge, and other developmental connectors are more thread-like.

This does not yet license a canonical frame layer beyond what `patterns/handle-schema.md` already reserves. It does license more careful language in living notes about whether a given object is tracking a line or reading through a lens.

## Held below promotion

The following should stay live but unpromoted:

- a full imported operation taxonomy such as `translate / pressurize / trace / synthesize`
- a mandatory `Candid` field for ordinary translation or review artifacts
- a repo-wide watchpoint / hook system before one local pilot proves useful more than once
- treating `pkm-religion-risk` and the Schuon limit as literally the same claim rather than related warnings
- treating the Factory / reaction-loop framing as the central description of `bildung-2.0`, where source resistance remains the primary feedback mechanism
- whole-repo movement scans or "state of bildung" reports as a standing artifact
- promotion of a formal cross-project frame layer before repeated frame-reads across anchors and projects show both usefulness and blind spots

## Practical use

Use this note when:

- deciding whether a recurring output type now deserves an artifact-specific handoff
- deciding whether a local apparatus should stay project-local or receive a bounded promotion scan
- deciding whether a proposed memory / hook / trigger convention belongs in one subproject ledger first rather than in `patterns/`
- deciding whether a synthesis object is functioning as a thread, a frame, or an unstable mixture of both
- checking whether a Symbiotic-Vault borrowing is operating-convention help or covert ontology import

Do not use this note as authorization for a broader vaultification of `bildung-2.0`.
