## Promoted claims

### 1. Controlled probe is a real missing mode between "located" and "full encounter"

This is the strongest surviving Warburg-side contribution.

The repo already has:

- a strong rule against importing modern concepts without encounter
- a live queue in `sources/modern/README.md`
- a trigger discipline that only promotes modern encounters when live work generates pressure

What the Warburg chat adds is a useful intermediate reading mode:

- not full encounter
- not mere filing
- one chapter or bounded slice
- one explicit question
- one note on whether the source resisted, taught, or merely confirmed

This should stay a living-layer practice, not a new canonical source ontology.

### 2. Methodological principles and current-model observations should be conceptually separated

This survives in a weaker form than the chat stated.

The real point is:

- "evidence before framework" is not the same kind of thing as "current LLMs over-taxonomize"
- one is a project method claim
- the other is an empirical note about a moving class of collaborators

That does not yet force a file split. But it is a useful reading rule whenever repo guidance is being revised.

### 3. A lightweight cross-project review rhythm may be useful, but only as observation support

The best weakened form of the ligature / metabolic idea is:

- an occasional, bounded reread of recent journals or notes across subprojects
- solely to notice missed edges or repeated cross-project pressure
- with no planning backlog and no new standing layer generated by default

This is much smaller than the full Warburg architecture. It is closer to an occasional review habit than to a new object type.

## Held as productive prompts, not findings

The following remain live but below the promotion line:

- a positive quarantine label for modern sources that are clearly important but not yet encountered
- a simple benchmark habit for testing new models on known-difficult passages
- lightweight `[method]` or `[process]` markers if recurring process findings start getting lost
- a very small ligature-style note format if cross-project review becomes regular enough to need one

These may become useful. None of them has yet earned canonical status or a repo-wide refactor.

## Rejected or not yet earned from this scope

The following should stay out of promotion from this note:

- quantitative pulse scripts as structural decision instruments
- `pulse-log.yaml`, manifests, or other monitoring-first infrastructure
- a full ligature / metabolic-review architecture
- `HORIZONS.md` or a capability-scaled task ontology
- two-tier cheap-model / expensive-model architecture as current process
- strong claims that session architecture will soon be obsolete because better models are coming
- treating the handle system as if it already solves model migration on its own

These are either too speculative, too tooling-first, or too tied to unstable assumptions about model capability.

## What survives only in weaker form

Several chat claims are not false so much as too large.

Weaker surviving versions:

- not "quarantine is a new status the repo needs now" but "some sources may need controlled-probe treatment before full encounter"
- not "split `CLAUDE.md` into `METHODOLOGY.md` and `MODEL-NOTES.md` now" but "keep method claims and model-behavior claims conceptually distinct when revising guidance"
- not "run ligature passes and metabolic reviews as standing rhythms" but "if cross-project edges are repeatedly being missed, test one small review pass"

## Use rule

Use this note when:

- deciding how much of the Warburg chat's method layer is actually worth retaining
- checking whether a modern-source handling proposal is a real gap or just a more elaborate name for existing `sources/modern/` discipline
- resisting tooling-first or future-model-first expansion

Do not use this note as authority to add new infrastructure by itself.
