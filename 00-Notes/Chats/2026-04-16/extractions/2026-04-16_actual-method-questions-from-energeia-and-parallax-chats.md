## Core question

Can the repo make a stronger, more auditable use of multiple passes, multiple languages, and predecessor depth without collapsing into free atmospheric synthesis?

## Question stack

1. Is there a real method in `stereoscopic translation` / `parallax` work, or only an attractive metaphor?

2. If multiple passes are legitimate, what makes their differences informative rather than just noise?

3. What would have to be recorded for pass-to-pass comparison to count as evidence?

4. Can predecessor texts become instrumented in a way that sharpens selective descent without building a KG-like ontology?

## Strong survivors

### 1. `Stereoscopic / parallax` translation is a real candidate method

The Aristotle chat contributes a bounded and intelligible method claim:

- choose target languages with orthogonal failure modes
- read the *strain* across the targets
- treat the strain as hypothesis-generation about what the source is doing
- do not treat the output itself as a settled philological finding

This survives because the chat also names the right cautions:

- target-language competence matters
- the method is heuristic, not evidential
- the result must be audited back against primary text and specialist work

### 2. `Parallax` can be generalized beyond language, but only under controls

The strongest late move in the creative-potential / dynamic-potential cluster is not the grand architecture. It is the claim that parallax can run across:

- languages
- models
- prompting techniques
- genealogical-depth settings

This is only useful if three missing controls are added:

- pass-comparison metadata
- a hermeneutics of how to read deltas
- real primary-work depth at the predecessor levels being loaded

Without those, multiple passes are not measurements; they are just multiple outputs.

### 3. `Predecessor instrumentation` is a live pressure, not yet a settled move

The creative-potential chat gives the cleanest formulation so far of what instrumented predecessors would actually be for:

- selective descent, not full predecessor loading
- dereferenceable pressure-bearing entries, not raw library access
- forward provenance across repos, not historical influence ontology

This feels real enough to keep visible, but it still wants one actual trial before it earns either architecture or a stronger method note.

### 4. `v1 failures are data` now has a sharper test

The strongest part of the dynamic-potential chat is its refusal to let `it's just v1` function as a permanent shield.

The practical survivors are:

- a `v1 lessons` file
- one explicit `what I would do differently` audit
- one real adversarial external contact

That is the cleanest operational test yet of whether the repo is actually converting failure into data rather than only saying it does.
