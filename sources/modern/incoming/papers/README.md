# Scientific Paper Intake

Purpose: keep scientific papers available for bounded comparison and reference
extraction without silently promoting every article PDF into a full modern
encounter.

This lane is for papers that matter because they pressure, test, or triangulate
an existing source question. It is not a general article library.

## Status Lanes

- `reference extraction`: machine text extracted from a PDF for search and
  quick lookup. It is not corrected text and is not citation authority.
- `control case`: a short note that says what live claim the paper tests. These
  usually belong in `../under-pressure/`, with the raw extraction here.
- `encounter candidate`: a paper that may deserve passage-level reading if a
  live project reaches for it again.
- `encounter`: promoted only when source work changes because of the paper.

## Reference Texts

Raw extractions live in `reference-text/` and use the suffix `.raw.txt`.

Tracked paper PDFs live in `source-pdfs/`.

Rules:

- the PDF remains the authority
- record the PDF path in any note that uses the extraction
- record what was actually checked from the PDF
- do not treat abstract/keyword agreement as full source contact
- do not clean, quote, or promote the paper unless the live work demands it

## When to Promote

Promotion requires more than topical relevance. A scientific paper should move
out of this lane only if it does one of the following:

- clarifies a live translation or source-reading problem
- acts as a strong control case for a claim already being made
- forces a correction to an existing note, glossary pressure, or source queue
- becomes part of a repeated cross-source pattern rather than a one-off echo

Until then, it can remain searchable reference material here plus a bounded
note in `../under-pressure/` when the comparison itself matters.
